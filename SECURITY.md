# Security Standards

> 本文档说明 Claude Skills Hub 的安全审查标准。所有收录项目均须通过以下全部检查。

## 强制安全检查（任意一项不通过即拒绝收录）

| 检查项 | 标准 |
|--------|------|
| 许可证 | 必须有明确的开源 License（MIT、Apache 2.0、CC-BY 系列等） |
| 代码可读性 | 无混淆/最小化代码（源码须可读），无隐藏恶意载荷 |
| 凭据安全 | 无硬编码 API key、密码或令牌；敏感配置须使用环境变量 |
| 外部域名 | 不得向不明域名发送请求；若有外部调用须明确记录并可关闭 |
| 数据收集 | 不得在未明确披露且用户未主动同意的情况下收集用户数据 |
| 危险代码 | 无未经说明的 `eval()`、`exec()` 或动态代码执行 |
| 依赖漏洞 | `package.json` / `requirements.txt` / `pyproject.toml` 中无已知 CVE 高危依赖 |
| 维护状态 | 仓库近 2 年内须有更新，或无悬而未决的安全 issue |
| README | 必须有说明文档 |

## 评分权重（固定，不得更改）

| 维度 | 权重 | 说明 |
|------|------|------|
| **安全** | **40%（固定）** | 通过/失败二元判断，失败即取消资格 |
| Stars | 最高 35 分 | ≤29=5, 30-99=15, 100-499=25, 500+=35 |
| 实用价值 | 最高 40 分 | 实际工作流价值、维护活跃度、文档质量 |
| 代码质量 | 最高 25 分 | 结构、测试覆盖、可读性 |

> **不可变规则**：安全检查的 40% 权重永不降低，安全不通过的项目永久取消资格，不论 Stars 多高或功能多实用。

## 安全披露

发现收录项目存在安全漏洞，请通过仓库 Issue 报告，并在标题中注明 `[SECURITY]`。
