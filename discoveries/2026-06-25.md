## 详细说明

### 1. Caveman
- **仓库**: https://github.com/JuliusBrussee/caveman
- **Stars**: 76,622 ⭐
- **最后更新**: 2026-06-12
- **分类**: skill
- **安全评级**: ✅ 安全（纯 Markdown SKILL.md，无代码执行，无网络请求）
- **功能简介**: 让 Claude Code（以及 Codex、Gemini、Cursor 等 30+ agent）像穴居人一样简洁作答，削减约 75% 的输出 token，同时保留完整技术精度。
- **触发方式**: 安装后在任意对话中通过 `/caveman` 调用，或将 SKILL.md 复制到 `~/.claude/skills/caveman/`
- **安装方法**: `git clone https://github.com/JuliusBrussee/caveman ~/.claude/skills/caveman`
- **适用场景**: 对话量大、关注 token 成本的开发者；需要快速迭代的 CI/自动化场景
- **注意事项**: 仅影响输出表达风格，不影响代码逻辑准确性

### 2. Headroom
- **仓库**: https://github.com/headroomlabs-ai/headroom
- **Stars**: 50,039 ⭐
- **最后更新**: 2026-06-25
- **分类**: library + MCP server
- **安全评级**: ✅ 安全（Apache 许可，CI 验证，本地优先，可逆压缩）
- **功能简介**: AI agent 的上下文压缩中间层，支持 6 种压缩算法，可将 tool 输出、日志、文件、RAG 块在送入 LLM 前压缩 60–95%，同时保持语义完整性。
- **触发方式**: 作为 Python/npm 库调用，或通过 MCP server 集成
- **安装方法**: `pip install headroom-ai` 或 `npm install headroom-ai`
- **适用场景**: 处理大型代码库、长日志、海量 RAG 数据的 agent；上下文窗口接近上限的场景
- **注意事项**: 压缩为有损操作（语义保留），极端边界情况可能丢失细节
