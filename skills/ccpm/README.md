# CCPM — The Project Manager Agent

> Spec-driven development for AI agents — ship better using PRDs, GitHub Issues, and parallel agents.

CCPM gives your AI agent a structured PM brain: turn ideas into PRDs, PRDs into epics, epics into GitHub issues, and issues into production code — with full traceability at every step.

**Works with:** Claude Code, Codex, OpenCode, Factory, Amp, Cursor, and more.

## The Workflow

```
Idea → PRD → Epics → GitHub Issues → Parallel Agent Execution → Production Code
```

## Core Principles

- **No vibe coding** — every change traceable to a GitHub issue
- **Context preserved** — issues survive session boundaries
- **Parallel execution** — multiple agents work non-conflicting tasks simultaneously
- **GitHub Issues as source of truth** — team-readable, AI-readable, versioned

## Key Features

- **PRD Creation** — guided brainstorming → structured Product Requirements Document
- **Epic Planning** — PRDs decomposed into logical implementation epics
- **Task Decomposition** — epics broken into atomic GitHub issues with acceptance criteria
- **GitHub Sync** — bidirectional issue tracking
- **Parallel Execution** — multiple agents on non-conflicting tasks
- **Progress Visibility** — kanban board view via GitHub Projects
